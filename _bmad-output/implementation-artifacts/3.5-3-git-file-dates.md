# Story 3.5.3: Git File Dates Collection

Status: done

## Story

As a **developer**,
I want Seshat to collect last git commit date for each file during scan,
so that convention trend detection can determine Rising/Stable/Declining. (FR63, ADR-24)

## Acceptance Criteria

1. `gix` crate added as dependency to `seshat-scanner`
2. During scan: single git commit walk builds `HashMap<PathBuf, i64>` of last modification timestamps per file
3. `files_ir` table has new nullable column `last_commit_date INTEGER` via migration
4. `ProjectFile` struct (or parallel storage) carries `last_commit_date` through the pipeline
5. Files not in git (new, untracked) have `last_commit_date = NULL`
6. Non-git directories (no `.git/`) skip date collection gracefully — no error, all dates `NULL`
7. Performance: single commit walk, not per-file `git log` — O(commits) not O(files)
8. `cargo test --workspace` passes
9. `cargo clippy --all-targets -- -D warnings` passes

## Dev Notes

### gix vs git2 vs shelling out

| Approach | Pros | Cons |
|----------|------|------|
| **gix** | Pure Rust, no C deps, actively maintained | Large API surface, commit walking can be complex |
| **git2** (libgit2) | Well-known API, simpler commit walk | C dependency (libgit2), adds ~2MB to binary |
| **Shell out to `git`** | Simplest, guaranteed correct | Requires git installed, spawns process, parsing overhead |

**Decision: gix** — aligns with zero-C-dependency philosophy. If the commit walk API proves too complex, fall back to shelling out to `git log` as pragmatic alternative and document it.

### gix Commit Walk Pattern

```rust
use gix::prelude::*;

let repo = gix::open(repo_root)?;
let head = repo.head_id()?;
let mut dates: HashMap<PathBuf, i64> = HashMap::new();

// Walk commits from HEAD
for commit_info in head.ancestors().all()? {
    let commit_info = commit_info?;
    let commit = commit_info.object()?;
    let time = commit.time()?.seconds;

    // Get tree diff to parent
    let tree = commit.tree()?;
    let parent_tree = commit.parent_ids().next()
        .and_then(|pid| pid.object().ok()?.tree().ok());

    // Diff trees to find changed files
    // ... insert files not already in map with this commit's timestamp
}
```

Note: The exact `gix` API may vary — check docs for `0.68.x`. The key principle is: walk commits newest-to-oldest, first occurrence of a file = its latest commit date.

### Architecture Compliance

- `git_dates.rs` in `seshat-scanner` — scanner owns file discovery and metadata
- Storage migration in `seshat-storage` — storage owns schema
- `ProjectFile` in `seshat-core` NOT modified (Option B) — IR stays pure code structure
- `gix` is a "big" dependency but it's pure Rust and tree-shakes well with feature flags

### Performance Considerations

- Single commit walk: O(total_commits) — for a 10k-commit repo, this is <1 second
- HashMap pre-allocation: `HashMap::with_capacity(discovered_files.len())`
- Only walk until all discovered files have dates (optimization: track remaining files, stop early when all found)
- For very large repos (100k+ commits): consider limiting walk depth (e.g., last 2 years) with a config option
