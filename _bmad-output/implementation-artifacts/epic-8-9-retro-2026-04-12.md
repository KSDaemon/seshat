## Epic 8: Built-in Embeddings & Semantic Search Quality

### Summary

- **Stories:** 8.1 (Built-in Embedding Provider), 8.2 (Rich Embedding Text Context)
- **Outcome:** `fastembed-rs` (all-MiniLM-L6-v2, 384-dim) integrated. HTTP providers (Ollama, OpenAI) removed. Zero-config embedding out of the box.
- **Deferred:** Code review findings D20-D22 (body snippet edge cases, type labels, dimension consistency)

### What Went Well

- Zero-config approach via `fastembed-rs` — no external services required, perfect for a dev tool
- `--no-default-features` compile-time opt-out — correct architectural safety valve
- Rich embedding text format (`{vis}{async}fn {name}({params}) in {path}\n{body}\nuses: {imports}`) is logical and reproducible
- HTTP provider removal simplified the codebase significantly

### What Didn't Go Well

- Code review findings D20-D22 were captured as deferred but not closed — carried as technical debt
- No `8.1-*.md` / `8.2-*.md` story files in `implementation-artifacts/` — Epic 8 was implemented via BMad workflow (not Ralph), and artifacts were not saved consistently. Creates a traceability gap.

### Technical Debt Carried Forward

| Item | Description | Priority |
|------|-------------|----------|
| D20 | Body snippet edge cases in embedding text | Medium |
| D21 | Type label consistency in embedding output | Medium |
| D22 | Embedding dimension consistency checks | Medium |

## Epic 9: CLI Utilities — Init

### Summary

- **Stories:** 9.1 (`seshat init` with auto-detection)
- **Outcome:** `seshat init` detects Claude Code, Claude Desktop, OpenCode, Cursor. Smart scope (project vs global), auto-patch with timestamped backup, dry-run mode, JSONC snippet-only safety.
- **Tests:** 158 unit + 3 integration tests

### Commit Pattern Analysis

```
c054346  feat(cli): implement seshat init command
0d1760c  fix(cli/init): code review fixes + smart scope logic
84efb04  fix(cli/init): use XDG ~/.config/opencode for OpenCode global config
8184698  fix(cli/init): use claude mcp add CLI for Claude Code, fix scope mapping
```

3 fix commits after the main implementation — signal of incomplete upfront research.

### What Went Well

- 158 unit tests + 3 integration tests — solid coverage for a stateful CLI command
- Smart scope logic: check for existing project-level config before falling back to global
- JSONC safety: show snippet only, never auto-patch files with comments (correct tradeoff)
- Clean terminal UX: `── copy ──` blocks, `[y/N]` prompt, timestamped backups

### What Didn't Go Well

- **Scope logic** was not fully designed in the story AC — discovered and fixed post-implementation
- **OpenCode XDG path** (`~/.config/opencode`) not verified before coding — wrong path used initially
- **Claude Code `claude mcp add` CLI** — discovered only after implementing direct JSON patching. Claude Code has a dedicated CLI for MCP management; direct JSON patching would have been fragile and potentially broken by future Claude Code updates. This was a fundamental approach change caught late.

### Root Cause

Insufficient research into client-specific APIs before writing code. The story went into implementation without a spike on "how does each client actually want to be configured".

---

## Lessons Learned (Both Epics)

### L1 — Spike before CLI integration stories (HIGH)

For any story touching external CLI tools or client APIs, add an explicit research task before implementation:
- Check for dedicated CLI commands (e.g., `claude mcp add`)
- Verify actual config file paths on each OS
- Read client changelogs for config format stability

**Apply from:** Epic 10 onwards

### L2 — Save story artifacts for all workflow types (MEDIUM)

BMad Dev Story workflow must produce `X.Y-story-name.md` in `implementation-artifacts/`, same as Ralph Loop. Epic 8 has no story files — traceability is lost.

**Apply from:** Epic 10 onwards

### L3 — Close deferred code review items before next epic (MEDIUM)

Code review findings marked as "deferred" must be tracked explicitly (e.g., in sprint-status or a debt register) and resolved within 1-2 epics. D20-D22 from Epic 8 are still open.

**Apply from:** Address D20-D22 in Epic 10 or as a pre-epic task
