# Story 9.1: `seshat init` — MCP Config Generator

Status: done

## Story

As a **developer**,
I want `seshat init` to detect my AI coding clients and generate (or auto-patch) MCP configurations,
so that I can connect Seshat to my AI tools in seconds without manually editing JSON files.

## Background & Design Decisions

### Supported Clients

| Client | Detection | Global Config | Project Config |
|--------|-----------|---------------|----------------|
| **Claude Code** (CLI) | `which claude` | `~/.claude/settings.json` → `mcpServers` | `.claude/settings.local.json` → `mcpServers` |
| **Claude Desktop** | app bundle exists at known path | `~/Library/Application Support/Claude/claude_desktop_config.json` → `mcpServers` | N/A |
| **OpenCode** | `which opencode` | `~/.config/opencode/opencode.json[c]` → `mcp` | `./opencode.json[c]` → `mcp` |
| **Cursor** | `which cursor` | `~/.cursor/mcp.json` → `mcpServers` | `.cursor/mcp.json` → `mcpServers` |

### MCP Entry Format per Client

**Claude Code / Claude Desktop / Cursor** (`mcpServers` schema):
```json
"seshat": {
  "command": "seshat",
  "args": ["serve"]
}
```

**OpenCode** (`mcp` schema — different structure!):
```json
"seshat": {
  "type": "local",
  "command": ["seshat", "serve"],
  "enabled": true
}
```

### JSONC Handling

OpenCode supports both `opencode.json` (JSON) and `opencode.jsonc` (JSON with comments).

Priority rules when both exist in same directory:
- `opencode.jsonc` takes precedence (opencode reads it first)
- If only `opencode.json` exists → JSON mode (can auto-patch)
- If `opencode.jsonc` exists → JSONC mode (show snippet only, never auto-patch)
- If neither exists → show snippet, tell user where to create the file

**Why not auto-patch JSONC:** Parsing and round-tripping JSONC preserving all comments
and formatting is non-trivial. The risk of silently destroying comments outweighs the benefit.
We clearly explain this to the user.

### Auto-patch Flow (JSON only)

1. Detect config file, parse JSON
2. Check if `seshat` key already exists in `mcpServers`/`mcp` section
3. If already configured → show "Already configured ✓", skip
4. Show the snippet to be added
5. Ask `[y/N] Auto-add to {path}?`
6. If y: write backup → merge entry → write updated JSON

### Backup Format

`{original_filename}.seshat-backup.{unix_timestamp}`

Example: `settings.json.seshat-backup.1744492800`

Backup is created in the same directory as the original file.
Backup is mentioned in output so user knows where it is.

### Scope: `--project` flag

- Default (no flag): operate on **global** config files
- `--project`: operate on **project-level** config files (in CWD or git root)
- Both scopes shown if `--project` is passed and project-level configs exist

Project-level config locations (searched in CWD, then git root):
- Claude Code: `.claude/settings.local.json`
- OpenCode: `./opencode.json` or `./opencode.jsonc`
- Cursor: `.cursor/mcp.json`

### `seshat init <client>` — Explicit Client

When client name given explicitly, skip detection and target that client only.
Supported names: `claude-code`, `claude-desktop`, `opencode`, `cursor`.
Unknown client name → error with list of supported names.

## Acceptance Criteria

### AC-1: Client Detection

**Given** no argument provided
**When** `seshat init` runs
**Then** the following clients are detected by checking PATH and known app locations:
- `claude` in PATH → Claude Code
- `~/Library/Application Support/Claude/` exists (macOS) → Claude Desktop
- `opencode` in PATH → OpenCode
- `cursor` in PATH → Cursor

**And** detected clients listed at top:
```
Detected AI coding clients:
  ✓ claude — Claude Code
  ✓ opencode — OpenCode
```

**And** if no clients detected:
```
No AI coding clients detected in PATH.

Supported clients: claude-code, claude-desktop, opencode, cursor
Run `seshat init <client>` to generate config for a specific client.
```

### AC-2: Claude Code — JSON auto-patch

**Given** Claude Code detected and `~/.claude/settings.json` exists (JSON, no comments)
**When** `seshat init` runs
**Then** config file parsed and checked for existing `seshat` entry in `mcpServers`

**If seshat NOT already in config:**
```
── Claude Code ──────────────────────────────────────────────

  Config: ~/.claude/settings.json
  Seshat is not configured. Add to "mcpServers":

  ── copy ─────────────────────────────────────────────────────
    "seshat": {
      "command": "seshat",
      "args": ["serve"]
    }
  ─────────────────────────────────────────────────────────────

  Auto-add? [y/N]
```

Rendering rules for the snippet block:
- Top line `── copy ──────` and bottom line `──────────────` are printed dim/grey (color) or plain dashes (NO_COLOR)
- The JSON lines between them are printed in default terminal color (uncolored, copyable as-is)
- No `│` characters on any line — user selects only the JSON lines between the two horizontal rules
- The word "copy" in the top rule makes it obvious which block to select

**If user presses y:**
- Backup created: `~/.claude/settings.json.seshat-backup.{timestamp}`
- `mcpServers.seshat` key added (creating `mcpServers` object if missing)
- File written with 2-space indentation, preserving existing structure
- Output:
```
  ✓ Backup saved: ~/.claude/settings.json.seshat-backup.1744492800
  ✓ Updated ~/.claude/settings.json
```

**If user presses n (or Enter):**
```
  Skipped. Add the snippet above manually.
```

**If seshat ALREADY in config:**
```
── Claude Code ──────────────────────────────────────────────

  Config: ~/.claude/settings.json
  ✓ Already configured.
```

### AC-3: OpenCode — JSONC mode (no auto-patch)

**Given** OpenCode detected and `opencode.jsonc` found (in global or project dir)
**When** `seshat init` runs
**Then**:
```
── OpenCode ─────────────────────────────────────────────────

  Config: ~/.config/opencode/opencode.jsonc
  Format: JSONC (contains comments — auto-patch not supported)

  Add to "mcp" section manually:

  ── copy ─────────────────────────────────────────────────────
    "seshat": {
      "type": "local",
      "command": ["seshat", "serve"],
      "enabled": true
    }
  ─────────────────────────────────────────────────────────────

```

**And** if seshat already present in the JSONC file (detected via text search for `"seshat"`):
```
  Config: ~/.config/opencode/opencode.jsonc
  ✓ Already configured (detected in JSONC — verify manually).
```
Note: JSONC already-configured check uses simple string search for `"seshat"`, not parsing.

### AC-4: OpenCode — JSON mode (auto-patch available)

**Given** OpenCode detected and only `opencode.json` found (no `opencode.jsonc` in same dir)
**When** `seshat init` runs
**Then** auto-patch flow same as AC-2 but:
- Targets `mcp` key instead of `mcpServers`
- Uses OpenCode snippet format (`type`, `command` as array, `enabled`)

### AC-5: JSONC + JSON both exist (opencode)

**Given** both `opencode.jsonc` and `opencode.json` exist in same directory
**When** `seshat init` runs
**Then** `opencode.jsonc` takes precedence (JSONC mode, no auto-patch)
**And** output notes: `Note: opencode.json also exists but opencode.jsonc takes precedence.`

### AC-6: Config file does not exist

**Given** Claude Code detected but `~/.claude/settings.json` does not exist
**When** `seshat init` runs
**Then**:
```
── Claude Code ──────────────────────────────────────────────

  Config not found: ~/.claude/settings.json
  Will create new file with:

  ── copy ─────────────────────────────────────────────────────
    {
      "mcpServers": {
        "seshat": {
          "command": "seshat",
          "args": ["serve"]
        }
      }
    }
  ─────────────────────────────────────────────────────────────

  Create file? [y/N]
```
**If y:** Parent directory created if needed, file written. No backup needed (new file).

**Given** `seshat init --dry-run`

**And** `[y/N]` prompts show as `[dry-run — no changes]` and are skipped

### AC-11: Output formatting

**And** all output uses existing `format.rs` utilities:
- Section headers via `print_section_header()` (box-drawing `──`)
- JSON snippets: **no border characters** — 4-space indent, blank line above and below

- Colors: `✓` green, `✗` red, warnings yellow, `NO_COLOR` respected

- Config file parse error (malformed JSON) → error with hint to fix manually
- File write permission denied → error with path + hint
- Backup write failure → abort, do not patch original

## Implementation Notes

### JSON merge strategy

Use `serde_json::Value` for reading/writing. Preserve all existing keys. Only add/update
`mcpServers.seshat` (or `mcp.seshat`). Use `serde_json::to_string_pretty` for output.
Do NOT use any JSONC-aware parser — if file has comments, treat as JSONC and skip auto-patch.

### JSONC detection

A file is JSONC if:
1. Extension is `.jsonc`, OR
2. Extension is `.json` but `serde_json::from_str` fails to parse it (has comments)

This means a mis-named `.json` file with comments is also handled safely (we'd try to parse,
fail, fall back to JSONC mode with snippet-only display).

### Claude Desktop detection (macOS-only)

```rust
#[cfg(target_os = "macos")]
fn detect_claude_desktop() -> Option<PathBuf> {
    let path = dirs::home_dir()?.join(
        "Library/Application Support/Claude/claude_desktop_config.json"
    );
    path.exists().then_some(path)
}
```

### Snippet rendering

Do NOT use `format::print_bordered_box()` for JSON snippets — border characters (`│ ┌ └`) end up
in the clipboard and require manual cleanup before pasting.

Instead use `print_copy_block(lines: &[&str])` — a new helper in `format.rs`:

```
  ── copy ─────────────────────────────────────────────────────   ← dim/grey
    <line 1>                                                       ← default color, 4-space indent
    <line 2>
  ─────────────────────────────────────────────────────────────   ← dim/grey
```

- Top rule contains the word "copy" as a visual cue
- Both horizontal rules are dim/grey when color enabled, plain dashes when NO_COLOR
- JSON content lines are default terminal color — no extra styling
- Width: 60 chars total (consistent with existing section headers)
- User selects only the content lines between the two rules — no border chars in clipboard
