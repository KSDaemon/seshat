# Story 3.5.1: Unify Dependency Domain Taxonomy

Status: ready-for-dev

## Story

As a **developer**,
I want a single, consistent dependency domain taxonomy across scanner and detectors,
so that domain classification has one source of truth with no duplication or contradictions.

## Acceptance Criteria

1. Single `DependencyDomain` enum in `seshat-core` replaces both `DependencyDomain` (seshat-detectors, 8 variants) and `DependencyCategory` (seshat-scanner, 12 variants)
2. Unified enum covers all domains from both sources: Http, WebFramework, Logging, Testing, Validation, Serialization, Database, Cli, AsyncRuntime, Crypto, Utilities, Unknown
3. `seshat-scanner/src/manifest.rs` uses unified enum — old `DependencyCategory` enum deleted
4. `seshat-detectors/src/dependency_usage.rs` uses unified enum — old `DependencyDomain` enum deleted
5. `DeclaredDependency.category` field in manifest.rs uses new unified type
6. All `classify_*()` and `categorize_*_dep()` functions updated to return unified type
7. All existing tests pass with new enum (`cargo test --workspace`)
8. `cargo clippy --all-targets -- -D warnings` passes

## Dev Notes

### Current State (What Exists)

Two parallel classification systems with subtle differences:

| Aspect | Detector (`dependency_usage.rs`) | Scanner (`manifest.rs`) |
|--------|----------------------------------|------------------------|
| Enum name | `DependencyDomain` | `DependencyCategory` |
| Variants | 8 (no Crypto, WebFramework, Utilities, Unknown) | 12 |
| Async variant | `AsyncRuntime` | `Async` |
| actix-web | Http | WebFramework |
| express/flask | Http | WebFramework |

### Architecture Compliance

- New enum goes in `seshat-core` — the shared types crate (per ADR-25)
- Both `seshat-scanner` and `seshat-detectors` depend on `seshat-core` — no new dependency edges
- Follow existing pattern: `seshat-core/src/knowledge.rs` for `KnowledgeNature`/`KnowledgeWeight` enums

### Mapping Decision for Overlapping Packages

Standardize: web frameworks (`express`, `fastify`, `flask`, `django`, `actix-web`, `axum`, `rocket`) → `WebFramework`. HTTP clients (`reqwest`, `axios`, `httpx`, `requests`) → `Http`. This aligns with the scanner's richer taxonomy.
