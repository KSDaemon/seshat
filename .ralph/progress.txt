## Codebase Patterns
- Workspace dependencies are declared in root Cargo.toml `[workspace.dependencies]`, then crates reference them via `workspace = true`
- Config structs in seshat-core/src/config.rs use `#[serde(default, rename_all = "snake_case")]` and have explicit `Default` impls with unit tests
- seshat.example.toml documents all config fields as comments with defaults shown — aspirational fields may exist there before the Rust struct has them
- Pre-commit hook runs cargo clippy + cargo fmt; if it modifies files (e.g. Cargo.lock), the commit fails and needs retry with those files staged
- seshat-mcp crate has NO direct storage access — queries go through seshat-graph
- The binary entrypoint is seshat-bin which calls seshat-cli::run()
