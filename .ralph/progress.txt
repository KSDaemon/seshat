## Codebase Patterns
- Use `refinery::embed_migrations!("migrations")` in `db.rs` to embed SQL migrations at compile time
- `Database` struct wraps `Arc<Mutex<Connection>>` — use `db.connection().lock().unwrap()` to get write access
- Migration files go in `crates/seshat-storage/migrations/` with naming `V{N}__{description}.sql`
- Core types (NodeId, EdgeId, BranchId, KnowledgeNode, Edge, etc.) are in `seshat-core` — import via workspace dependency
- All ID types use newtype pattern: `NodeId(pub i64)`, `EdgeId(pub i64)`, `BranchId(pub String)`
- `StorageError` in `seshat-storage/src/error.rs` is the crate-level error type — add variants there for new failure modes
- Use `":memory:"` for in-memory SQLite in tests (fast, no cleanup needed for simple cases)
- Workspace uses `thiserror = "2"` (not v1) for error derive macros
- The `seshat-core` crate has a `test-helpers` feature for test utility functions
- Repository pattern: create `SqliteXxxRepository` structs that take `Arc<Mutex<Connection>>` via `new()` — clone from `db.connection()`
- Enums are stored as snake_case TEXT in SQLite (matching serde `rename_all = "snake_case"`) — use manual serialize/deserialize helpers, not Display
- Clippy enforces `-D warnings` — use function references (`row_to_node`) instead of closures (`|row| row_to_node(row)`)
- `serde_json` is needed in seshat-storage for ext_data/metadata JSON columns — add as workspace dependency
- Add `seshat-core = { workspace = true, features = ["test-helpers"] }` to `[dev-dependencies]` for test factory functions
- For BLOB columns (e.g., `ir_data`), use `serde_json::to_vec` / `serde_json::from_slice` to serialize/deserialize entire structs
- Use `ON CONFLICT ... DO UPDATE SET` for upsert operations on tables with UNIQUE constraints
- Use `conn.unchecked_transaction()` for transactions within a `Mutex<Connection>` lock — avoids borrow issues
- `list_branches` gathers distinct branch_ids via UNION across nodes, edges, files_ir tables
- Current branch is stored in the `metadata` table under key `"current_branch"` — defaults to `"main"` if absent
