## Codebase Patterns
- Config module (AppConfig) lives in seshat-cli, not seshat-bin — bin stays thin
- build.rs for GIT_HASH lives in seshat-cli (same crate as clap types that use it)
- scan_project() has two variants: scan_project() (no progress) and scan_project_with_progress() (with callback)
- ScanProgress enum has 4 variants: Discovering, DiscoveryDone, Scanning, ScanningDone
- Convention detection runs AFTER scan pipeline: load files from DB → run_all_detectors → aggregate_findings
- DB path follows XDG convention: dirs::data_dir()/seshat/repos/{project_name}.db
- tracing-subscriber uses SESHAT_LOG env var, defaults to "warn" to avoid cluttering CLI output
- Use `eprintln!` for CLI status output (not println!) so it doesn't mix with potential JSON/structured stdout
- Pre-commit hooks run: trim whitespace, fix EOF, check toml, cargo fmt, cargo clippy
- cargo fmt may reformat imports (alphabetical) and assert! macros — always stage after fmt hook
- All format functions take `color: bool` parameter; caller passes `format::color_enabled()` once at top of command
- Verbosity enum: from_flags(verbose, quiet) → Quiet/Default/Verbose; quiet wins if both set
- Box-drawing chars (─, ▓, ░, ┌, ┐, │, └, ┘) are multi-byte UTF-8 — use .chars().count() not .len() for display width
- format.rs is the SINGLE module for all CLI output formatting — US-003/US-004 must use these helpers
