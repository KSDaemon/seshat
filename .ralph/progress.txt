  - On crates.io the crate is published as `self-replace` (dash); the Rust import is `use self_replace` (underscore) βÄî but with `self-replace.workspace = true` in the per-crate `Cargo.toml`, the call site is just `self_replace::self_replace(...)` with no extra `use` line, leveraging the same path-vs-import convention as `tracing-subscriber` / `use tracing_subscriber`.
  - When refactoring an error-mapping helper to add a `cfg(windows)` branch, branch BOTH the user-facing eprintln string AND the machine-readable `reason: String` field of `CliError::CommandFailed`. Tests that introspect the error reason then have a stable assertion target (`reason.contains("Administrator")`) without piping stderr.
  - The "simulate read-only target βÜ£ admin hint" AC is most cleanly satisfied by feeding a synthetic `io::Error::from(io::ErrorKind::PermissionDenied)` through `map_replace_error` and asserting on the resulting `CliError`. Building an actual read-only target file on Windows is brittle (ACL behavior varies between local admin / non-admin runners) and irrelevant βÄî the contract under test is *the mapping*, not the kernel-level permission probe.
  - `io::Error::other(msg)` (stable since Rust 1.74, well below our 1.85 MSRV) is the idiomatic way to synthesize a non-PermissionDenied io error in tests; cleaner than `io::Error::new(io::ErrorKind::Other, msg)`.
  - Removed `replace_binary_on_same_filesystem` is **not** replaceable by an integration test βÄî the same self-replace-corrupts-test-runner constraint applies. Coverage of the `Ok(())` arm of the new `replace_binary` will land via the Windows CI run in US-008 (where the real release-flow integration test in US-007 exercises the path end-to-end against a mocked GitHub release).
  - `cfg(windows)` is the right gate for the new admin-hint and Windows cargo-install tests (rather than `cfg(target_os = "windows")`); both forms are equivalent here because the predicate is only on OS, but `cfg(windows)` is the canonical short form and matches the rest of `update.rs`.
  - `let _ = fs::remove_dir_all(temp_dir);` on the `Err` path is redundant with the `tempfile::TempDir` Drop impl that runs when `run_self_update` exits, but kept to match the eager-cleanup convention used by every other error path in this file (consistency > strict minimalism here).
