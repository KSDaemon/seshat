## Codebase Patterns
- CodeEvidence struct is defined in `crates/seshat-core/src/detector_result.rs` and constructed in ~13 detector files across seshat-detectors and seshat-graph
- All CodeEvidence constructions follow the pattern: `file`, `line`, `end_line`, `snippet` fields in order
- Some constructions use multi-line `snippet:` values (e.g., `format!()`, `if/else` expressions) ‚ÄĒ simple regex may miss these
- ConventionFinding wraps Vec<CodeEvidence> ‚ÄĒ changes to CodeEvidence propagate to all detectors
- `#[serde(default)]` on a field means old JSON without that field deserializes with the type's Default (0 for usize)
- Pre-commit hooks run `cargo fmt` and `cargo clippy` ‚ÄĒ always re-stage after first commit attempt
- Integration tests for detectors follow pattern: build a `ProjectFile` using the detector's `make_*_file` helpers, call `detect_with_source(&file, &source)`, assert on evidence fields ‚ÄĒ no mocking needed
- `detect_with_source` fills empty snippets from source with `EVIDENCE_CONTEXT_BEFORE=2` lines of leading context; `snippet_start_line` is set to the actual start line
- Pre-populated (non-empty) snippets in `FunctionCall.snippet` are preserved by `detect_with_source` (US-002 fix)
