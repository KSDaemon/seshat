## Codebase Patterns
- Use `gix::open(path)` + `repo.references()?.local_branches()` to list local git branches
- Use `entry.name().as_bstr().to_str()` to get branch name string (requires `gix::bstr::ByteSlice` trait in scope)
- Use `gix_ref::FullNameRef` via `entry.name()` to access full ref path (e.g., `refs/heads/feature-x`)
- Strip `refs/heads/` prefix manually to get short branch name
- `SqliteBranchRepository` has `list_branches()`, `delete_branch()`, `switch_branch()`, `create_snapshot()` methods
- `PROTECTED_BRANCHES` constant pattern: use `&[&str]` with `.contains(&name.as_str())` for safety checks
- Call `gc_branch_snapshots(&db, &repo_path)` in `run_serve` after branch handling, before MCP server start
- GC repo path resolution: use `auto_scan_project_root` if available, otherwise `find_git_root(cwd)` fallback
- `list_branches()` returns branches from data tables (nodes/edges/files_ir) ‚Äî branches are implicitly registered when data exists
- For GC tests: insert data via `SqliteFileIRRepository::upsert` first, then `create_snapshot` to register other branches
- Use `tokio::time::interval(Duration::from_secs(3600))` for periodic background tasks
