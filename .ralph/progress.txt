## 2026-05-02 23:10 (EEST) - US-001
- What was implemented:
  - Added fuzzy search/filter to TUI convention review wizard
  - `/` enters search mode, bottom row shows filter input bar with cursor
  - Typing filters conventions in real time by description field using case-insensitive substring + Levenshtein distance ≤ 2 fuzzy matching
  - Backspace removes last character; empty filter exits search mode
  - Enter locks filter (exits search mode, keeps filter active)
  - Esc clears filter and exits search mode
  - y/n/p/s keys blocked in search mode
  - Navigation (arrows/jk) walks only through filtered subset; counter reflects filtered count
  - Empty state shows "No matching conventions" when no matches
  - Locked filter shows header indicator `[filter: 'keyword']`
- Files changed:
  - `crates/seshat-cli/src/tui/app.rs`: Added search_mode, search_query, filter_locked, filtered_indices fields; levenshtein_distance() and fuzzy_match() utility functions; filtered_next/previous/current_index/total methods; push_search_char/pop_search_char/lock_filter/exit_search_mode/cancel_search
  - `crates/seshat-cli/src/tui/review_wizard.rs`: Updated handle_key for search mode dispatch, blocked y/n/p/s in search mode, Enter locks, Esc cancels, typing passes chars
  - `crates/seshat-cli/src/tui/widgets.rs`: Updated render() and ConventionCard struct with search bar rendering, locked filter header, empty state display, filtered count in header
- **Learnings for future iterations:**
  - The ConventionCard Widget struct uses borrowed references (&'a ConventionItem) — adding new fields requires updating all test initializers
  - Ratatui Layout::vertical returns areas into a dynamic-count layout, not destructuring — used index-based access for variable-length layout
  - The filtered_indices field must be public for widgets.rs to access it (can't use a getter in the Widget impl since ConventionCard doesn't hold an App reference)
