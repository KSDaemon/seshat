## Codebase Patterns
- Migrations live in `crates/seshat-storage/migrations/V<N>__<name>.sql`,
  embedded via `refinery::embed_migrations!`. Refinery's SQL files cannot
  deserialize postcard blobs ‚Äî for backfill that needs Rust, perform it in
  `Database::open` immediately after `migrations::runner().run(...)`, gated
  on a "table empty?" check so it stays idempotent across reopens.
- IR snippets must be built the same way wherever they are produced. Synthetic
  definition snippet helpers (function/type/export) live in
  `seshat-core::symbol_snippet` so `seshat-storage` (writer) and
  `seshat-graph` (reader) emit identical strings without one crate depending
  on the other.
- Repository traits live in `crates/seshat-storage/src/repository/mod.rs`;
  Sqlite impls live in sibling files. Re-export everything via
  `crates/seshat-storage/src/lib.rs` so downstream crates only import from the
  crate root.
- Per-file index writes use the `replace_file` pattern: `DELETE` rows for
  `(branch_id, file_path)` then `INSERT` the new set, all in one
  `unchecked_transaction()`. Keeps re-scans / hot-tier updates idempotent.
- Tests that need to seed `files_ir` directly with serialized IR can use
  `crate::ir_serialization::serialize_ir(&project_file)` and bypass the
  repository.

- **Learnings for future iterations:**
  - Refinery 0.9 SQL migrations cannot run Rust; backfill that touches
    serialized blobs has to live in `Database::open` after the migrator runs,
    not in the `.sql` file itself. The AC's "inside the migration" wording is
    satisfied by the upgrade-time backfill being part of the V13 step.
  - The `SymbolKind::Export` row stores `is_public = true` unconditionally
    because exports are by definition externally-reachable ‚Äî the IR
    `Export` struct has no `is_public` field.
  - Stale IR rows (older `ir_schema_version`) are skipped during backfill,
    matching how `get_file_hashes_by_branch` already excludes them. They will
    repopulate after the user runs `seshat scan`.
  - Wildcard imports filter is `name == "*" || name.starts_with("* as ")` ‚Äî
    the second guard catches the TS parser's `import * as foo from '‚Ä¶'`
    output which stores `["* as foo"]` in `Import.names`.
  - `cargo fmt --all -- --check` flagged trailing-method splits across
    several test lines in `symbol_index_repository.rs`. Run `cargo fmt --all`
    before committing.
