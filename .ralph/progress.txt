## Codebase Patterns
- Repository pattern: Trait in `repository/mod.rs`, impl in `repository/{name}_repository.rs`, struct wrapping `Arc<Mutex<Connection>>` with `new()` + `conn()` helper, `row_to_{name}()` private mapper function
- Migrations: `V{N}__{description}.sql` in `crates/seshat-storage/migrations/`, compiled at build time by `refinery::embed_migrations!("migrations")`
- Tests: `Database::open(":memory:")` for in-memory DB, test helper creates repo instance, no external dependencies
- Wiring: New repo modules declared in `repository/mod.rs` (mod + pub use), traits and types re-exported from `lib.rs`
- Pre-commit hook: runs `cargo fmt` then `cargo clippy`. If fmt modifies files, commit is rejected — stage the formatted files and retry
- Import ordering: `rusqlite::{params, Connection}` (alphabetical) is the canonical `cargo fmt` order
- DB path layout: root DB at `repos/{project_name}.db`, submodule DBs at `repos/{project_name}/{mount_path}.db`
- ScanConfig uses `exclude_submodules` (not include_submodules) — default false means submodules ARE scanned
- Root discovery ALWAYS excludes submodule dirs (they get their own DBs); `exclude_submodules` controls whether separate scans happen
- `detect_submodule_paths()` is pub in `seshat-scanner::discovery` and re-exported from `seshat-scanner::lib.rs`
- ScanProgress enum is in `seshat-scanner::orchestrator`, re-exported from `seshat-scanner::lib.rs`; new submodule variants added in US-003
- `get_submodule_commit_hash()` uses `gix::open()` + `head_commit()` — works with both `.git` files (submodule pointers) and `.git` directories
- CLI progress handling in `scan.rs` uses `RefCell<HashMap<String, ProgressBar>>` for per-submodule spinners keyed by mount path
- Submodule scan flow in `scan.rs::run_scan()`: detect submodules → scan each into own DB → run root scan → update submodules table + repo_metadata in root DB
- `resolve_submodule_db_path()` no longer has `#[allow(dead_code)]` — it's used by the submodule scan flow
- Submodule scan reuses `scan_project_with_progress()` with the submodule's abs path and dedicated DB — no new orchestrator logic needed
- After scanning each submodule: run convention detectors, persist conventions, write repo_metadata (parent_project, mount_path, file_count, convention_count, last_scan_time)
- Root DB submodules table: upsert pattern (try update first, insert if NotFound); stale entries removed when mount_path no longer in .gitmodules
- Root DB repo_metadata: project_name, file_count, convention_count, last_scan_time written after root scan completes
