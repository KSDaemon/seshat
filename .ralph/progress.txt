## Codebase Patterns
- All IR types (ProjectFile, Language, LanguageIR, etc.) live in seshat-core::ir — parsers import from seshat_core
- Documentation ingestion lives in seshat-scanner::documentation; uses serde_yaml (workspace dep) for OpenAPI YAML parsing
- Use `yaml_key()`, `yaml_get_str()`, `yaml_get_mapping()`, `yaml_get_seq()` helpers to avoid clippy needless_borrows warnings with serde_yaml::Mapping
- Documentation nodes use KnowledgeNature::Fact + KnowledgeWeight::Info with ext_data containing `"source": "documentation"` tag
- DocType::from_extension() detects .md → Markdown, .json → JsonSchema, .yaml/.yml → OpenApi
- JSON Schema detection: file must be a JSON object with `$schema`, `properties`, or (`type` + `title`)
- OpenAPI detection: YAML mapping with `openapi` or `swagger` top-level key
- Use `LanguageIR::Rust(RustIR::default())` etc. for empty language-specific IR
- Test helpers behind `test-helpers` feature flag in seshat-core; use via `[dev-dependencies] seshat-core = { workspace = true, features = ["test-helpers"] }`
- Workspace dependencies declared in root Cargo.toml under `[workspace.dependencies]`, crates reference them with `.workspace = true`
- ScanError in seshat-scanner::error is the shared error type for the scanning pipeline
- Parser trait + dispatch live in seshat-scanner::parser::mod.rs; each language parser is a submodule
- Content hashing (SHA-256 via sha2 crate) is done in shared code (parser::content_hash), not per-parser
- Graceful degradation: parse_file returns empty ProjectFile with tracing::warn on parser error
- File discovery lives in seshat-scanner::discovery; uses `ignore` crate WalkBuilder for .gitignore support
- Language::from_extension(ext) detects language from file extension; returns None for unrecognised
- ScanConfig (in seshat-core::config) has exclude_patterns and max_file_size_kb — used by discover_files
- Use `tempfile` crate (workspace dep) for filesystem tests with temp directories
- Tree-sitter parsers: use `tree-sitter = "0.25"` and `tree-sitter-rust = "0.24"`, `tree-sitter-typescript = "0.23"` (compatible set)
- Tree-sitter parser setup: `TsParser::new()` → `set_language(&tree_sitter_rust::LANGUAGE.into())` → `parse(source, None)`
- Tree-sitter node traversal: iterate `root.child_count()` with `root.child(i)`, match on `child.kind()`
- Node text: `node.utf8_text(source.as_bytes())` — always pass `&[u8]` not `&str`
- Visibility detection: look for `visibility_modifier` child node on items
- Derive attributes: `attribute_item` → `attribute` child → text starts with "derive" → parse parens content
- Trait impls: `impl_item` with `for` keyword distinguishes trait impls from inherent impls
- Integration test fixtures go in `crates/seshat-scanner/tests/fixtures/<language>_project/`
- DeriveUsage and TraitImpl are now re-exported from seshat-core (added to lib.rs pub use)
- tree-sitter-typescript exposes LANGUAGE_TYPESCRIPT and LANGUAGE_TSX — use TSX for .tsx files
- TypeScript AST: imports → `import_statement` with `import_clause` (containing `named_imports`, `identifier`, or `namespace_import`)
- TypeScript AST: exports → `export_statement` with children: `export_clause`, `default`, `function_declaration`, `class_declaration`, `lexical_declaration`, `*`
- TypeScript AST: decorators on exported classes appear as children of `export_statement`, not `class_declaration`
- TypeScript AST: type-only imports/exports have a `type` child node (not an attribute like in Rust)
- TypeScript AST: `enum_declaration` uses `identifier` not `type_identifier` for its name (unlike interface/class/type_alias)
- Pre-commit hooks run cargo fmt — if hook fails, re-add formatted files and make a new commit
- tree-sitter-javascript 0.23 exposes `LANGUAGE` constant (single grammar handles both JS and JSX)
- JavaScript AST is similar to TypeScript: `import_statement`, `export_statement`, `function_declaration`, `class_declaration`, `lexical_declaration`, `expression_statement`
- CommonJS patterns: `require('module')` → `call_expression` with `identifier` "require" and `arguments` containing `string`
- Destructured require: `const { a, b } = require('module')` → `variable_declarator` with `object_pattern` + `call_expression`
- `object_pattern` children: `shorthand_property_identifier_pattern` (for `{ a }`) and `pair_pattern` (for `{ a: b }`)
- `module.exports = ...` and `exports.foo = ...` are `expression_statement` → `assignment_expression` → `member_expression`
- Module system detection order: check destructured require before plain require (destructured has `object_pattern` which would fail the simple `identifier` check)
- tree-sitter-python 0.23 is compatible with tree-sitter 0.25 — exposes a single `LANGUAGE` constant
- Python AST: `async def` is a `function_definition` with an `async` keyword child (kind="async"), NOT a separate node type
- Python AST: `import_statement` has `dotted_name` or `aliased_import` children; `import_from_statement` has `relative_import` for relative imports
- Python AST: `decorated_definition` wraps `decorator` + `function_definition`/`class_definition` — decorators are children, not siblings
- Python AST: `__all__` is `expression_statement` → `assignment` with `list` RHS containing `string` children
- Python string nodes have `string_content` child for extracting inner text (no quotes)
- Type hint detection: check `type` child (return annotation), `typed_parameter`/`typed_default_parameter` in params, AND class body methods/annotated assignments
- Manifest analysis lives in seshat-scanner::manifest; uses `toml` and `serde_json` crates (both workspace deps)
- Cargo.toml deps can be `dep = "version"` or `dep = { version = "...", ... }` or `dep.workspace = true` — use serde Deserialize with HashMap<String, toml::Value>
- PEP 508 dependency specs: name ends at first non-alphanumeric/non-dash/non-underscore/non-dot character
- Python package names normalise hyphens to underscores for import matching
- Cross-referencing: Rust/Python normalise hyphens→underscores; JS matches exact name or subpath (`dep/path`)
- ScanError::ManifestError variant for manifest parsing errors
- Module structure lives in seshat-scanner::module_structure; `build_module_graph(root, parsed_files, branch_id)` returns `ModuleGraph`
- ModuleGraph stores `path_to_node_id` and `node_id_to_path` maps for O(1) queryability via `dependencies_of()` / `dependents_of()`
- Modules are detected by directory: each directory containing source files = one module (Fact node)
- Import resolution uses `build_import_target_map` which registers files by path, stem, Rust `::` style, Python dotted style, and directory-level targets
- DependsOn edges are deduplicated per (source_module, target_module) pair using BTreeSet
- PartOf edges only created when parent directory is also a detected module (has source files)
- `parse_file` returns `ProjectFile` directly (not Result) — graceful degradation by design
- IR serialization uses bincode with version prefix (ADR-16): first byte = IR_SCHEMA_VERSION, rest = bincode data
- `serialize_ir` / `deserialize_ir` in seshat-storage::ir_serialization — version mismatch returns StorageError::StaleIR
- bincode 1.x API: `bincode::serialize_into(&mut buf, &value)` and `bincode::deserialize(&data)` — uses serde derives
- file_ir_repository uses serialize_ir/deserialize_ir (not serde_json) for DB storage — bincode is more compact
- Scan orchestration lives in seshat-scanner::orchestrator; `scan_project(root, config, db)` returns `ScanResult`
- seshat-scanner depends on seshat-storage for DB access during orchestration (no circular dependency: scanner→storage→core)
- `build_module_graph` assigns placeholder NodeIds (sequential from 1); when persisting, must remap to real DB-assigned IDs via `NodeRepository::insert()` return value
- Edge remapping pattern: build HashMap<NodeId, NodeId> (placeholder→real), apply to all edges before inserting
- `ScanError::StorageError` variant wraps `seshat_storage::StorageError` with `#[from]` conversion
- Doc file discovery: JSON files must pass schema detection (has `$schema`, `properties`, or `type`+`title`); YAML must have `openapi`/`swagger` key
- `serde_yaml::Mapping::contains_key()` takes owned `Value`, not `&Value` — drop the `&` to avoid clippy needless_borrows warning
- Incremental scan: `get_file_hashes_by_branch` returns HashMap<String, String> (path→hash) — lightweight alternative to `get_by_branch` when only path+hash needed
- Incremental scan: module nodes/edges are fully rebuilt on each scan (delete old, insert new) rather than diffed — simpler and more reliable
- `delete_by_branch` on NodeRepository and EdgeRepository deletes all rows for a branch — used for module structure rebuild
- `scan_project` auto-detects incremental mode by checking `stored_hashes.is_empty()` — no separate function needed
- `IncrementalStats` tracks files_unchanged/changed/new/deleted; `ScanResult.incremental` is `Option<IncrementalStats>` (None = fresh scan)
- For incremental re-scan with unchanged files, use `get_by_branch` to load all IR from DB after upserts — needed for module graph which requires full parsed file set
- Backup logic lives in seshat-storage::backup; `backup_if_needed(db_path, config)` returns `Ok(true)` if backup created, `Ok(false)` if skipped
- BackupConfig in seshat-core::config: `enabled` (default true), `retention_count` (default 3), `interval_hours` (default 24)
- Backup filenames: `<db_filename>.<YYYY-MM-DD>` — date suffix enables sorted listing and interval checking
- Date calculation uses Howard Hinnant's civil_from_days / days_from_civil algorithms — no chrono dependency needed
- `StorageError::Io` has `#[from] std::io::Error` — use `map_err(StorageError::Io)` not `.into()` to avoid clippy useless_conversion
