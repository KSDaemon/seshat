## Codebase Patterns
- seshat-core re-exports all public types from lib.rs â€” use `seshat_core::TypeName` not `seshat_core::module::TypeName`
- Every crate has its own `error.rs` with `thiserror` derive macros
- Detectors only depend on `seshat-core` (for IR types) plus `rayon`, `tracing`, `thiserror` â€” no storage/scanner deps
- `ConventionFinding`, `DetectorResults`, `CodeEvidence` live in `seshat-core::detector_result` (re-exported from lib.rs)
- `DetectionConfig` thresholds: >0.85 Strong, 0.50-0.85 Moderate, 0.20-0.50 Weak, <0.20 Info (boundary values go to the lower tier)
- `Language::all()` returns all 4 supported languages as a static slice
- `LanguageIR` enum variants: `Rust(RustIR)`, `TypeScript(TypeScriptIR)`, `JavaScript(JavaScriptIR)`, `Python(PythonIR)`
- Test pattern: inline `#[cfg(test)] mod tests` at bottom of each file; use `make_project_file()` from test_helpers for quick ProjectFile construction
- Pipeline design (ADR-6): files processed in parallel via `rayon::par_iter()`, detectors run sequentially per file
- Failing detectors are caught via `std::panic::catch_unwind` and logged with `tracing::warn` â€” never crash the pipeline
- `#[tracing::instrument(skip_all)]` on public pipeline functions
- Workspace uses Rust edition 2024 (min 1.85)
- Detector pattern: create struct implementing `ConventionDetector`, register in `all_detectors()` in pipeline.rs, add `pub mod` to lib.rs
- `DependencyUsage` has `package`, `import_path`, `line` â€” `package` is the crate/package name, `import_path` is the full import path
- Domain classification: use match on package name per language to map to `DependencyDomain` enum
- Convention findings: `follows_convention: true` for canonical library usage, `false` for conflicts and dead deps
- Use `KnowledgeNature::Convention` for canonical library findings, `KnowledgeNature::Observation` for conflicts and dead deps
- Import classification: Rust (`std/core/alloc` â†’ Stdlib, `crate/super/self` â†’ Internal, rest â†’ External), TS/JS (relative `./`/`../` â†’ Internal, node builtins â†’ Stdlib, rest â†’ External), Python (known stdlib list â†’ Stdlib, relative `.` or `src/app/lib` prefix â†’ Internal, rest â†’ External)
- `Import` struct: `module` (path), `names` (imported symbols), `is_type_only` (TS type-only), `line` (source line)
- For grouping analysis, check monotonic non-decreasing order of ImportGroup enum values and line gaps >1 for blank-line separation
- `RustIR.error_types` populated by scanner for structs/enums with "Error" in name â€” use directly, no re-parsing needed
- `RustIR.derive_macros` has `DeriveUsage { type_name, derives, line }` â€” check `derives` for specific derive macro names
- `RustIR.trait_implementations` has `TraitImpl { trait_name, type_name, line }` â€” useful for detecting trait impls like `Error`, `From<T>`
- Use `DETECTOR_NAME` constant for detector name to keep it consistent across findings
