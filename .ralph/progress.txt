## Codebase Patterns
- Workspace monorepo: 10 crates under `crates/`; `seshat-bin` is the thin binary, `seshat-cli` is the CLI library
- CLI commands are organized as one module per file in `seshat-cli/src/`, declared in `lib.rs`, dispatched via `match cli.command` in `run()`
- Error handling: `thiserror`-based `CliError` enum, NOT `anyhow`
- Paths: `dirs` crate for XDG resolution; DBs at `$XDG_DATA_HOME/seshat/repos/`, cache at `$XDG_DATA_HOME/seshat/version-check.json`
- Testing: `#[cfg(test)] mod tests` inline unit tests; integration tests in `tests/` dirs using `assert_cmd` + `predicates`
- Serialization: `serde` + `serde_json` (already available workspace deps)
- Timestamps: `chrono` with `clock` feature (already available workspace dep)
- Commit hooks: `cargo fmt` and `cargo clippy` run via pre-commit
- Update commands: `seshat update --check` queries GitHub Releases API, uses `ureq` with 15s timeout
- Version comparison: custom semver parser (no external `semver` crate needed) — handles major.minor.patch with extra components
- Homebrew detection: checks if current_exe path contains `/Cellar/` (also resolves symlinks via canonicalize)
- Target triple mapping: `std::env::consts::ARCH` + `std::env::consts::OS` → target string (aarch64-apple-darwin, x86_64-apple-darwin, x86_64-unknown-linux-gnu, aarch64-unknown-linux-gnu)
- Update sub-commands: `run_update(check)` dispatches — `check=true` → `run_check()`, `check=false` → placeholder for future US-003
- `update` command is NOT suppressed from background notice (US-005 will handle that)
- Test pattern for network-dependent functions: use `_inner` variants that accept optional cache path, allowing isolation without mocking
