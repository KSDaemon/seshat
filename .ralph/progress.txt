## Codebase Patterns
- Workflow syntax: Use `${{ }}` template expressions in GitHub Actions YAML — they confuse static analyzers. Export them to shell variables before using to avoid shellcheck false positives
- Release workflow: release-plz job only runs on branch pushes (`refs/heads/main`) via `if:` condition; binary build and upload jobs only run on tag pushes (`startsWith(github.ref, 'refs/tags/')`)
- Sha256: Generate in a dedicated `generate-checksums` job (ubuntu-latest) after all platform builds complete, not inside per-platform jobs
- Quality: actionlint must return exit code 0; use `cargo check --workspace` for typecheck
- No CLAUDE.md files exist in this repository — use `.ralph/progress.txt` Codebase Patterns section instead
- install.sh is POSIX sh compatible (#!/bin/sh), not bash — avoid bash-specific syntax like [[ ]], arrays, or == for string comparison
- Shellcheck must pass with 0 warnings (0 errors). Run: `shellcheck install.sh`
- install.sh uses colors via escape codes defined as shell variables (RED, GREEN, YELLOW, NC) for consistent output formatting
- OS detection: uname -s maps to Linux/Darwin, uname -m maps to x86_64/aarch64. Concatenated as `{arch}-unknown-linux-gnu` or `{arch}-apple-darwin`
- WSL detection: check `/proc/version` for "microsoft" substring
- Checksums: sha256sum on Linux, `shasum -a 256` on macOS
- Download: prefers curl, falls back to wget
- GitHub API: use `grep '"tag_name"' | sed` to extract tag from JSON response
- install.sh lives at repo root, not in tools/
