## Codebase Patterns
- Use `refinery::embed_migrations!("migrations")` in `db.rs` to embed SQL migrations at compile time
- `Database` struct wraps `Arc<Mutex<Connection>>` ‚ÄĒ use `db.connection().lock().unwrap()` to get write access
- Migration files go in `crates/seshat-storage/migrations/` with naming `V{N}__{description}.sql`
- Core types (NodeId, EdgeId, BranchId, KnowledgeNode, Edge, etc.) are in `seshat-core` ‚ÄĒ import via workspace dependency
- All ID types use newtype pattern: `NodeId(pub i64)`, `EdgeId(pub i64)`, `BranchId(pub String)`
- `StorageError` in `seshat-storage/src/error.rs` is the crate-level error type ‚ÄĒ add variants there for new failure modes
- Use `":memory:"` for in-memory SQLite in tests (fast, no cleanup needed for simple cases)
- Workspace uses `thiserror = "2"` (not v1) for error derive macros
- The `seshat-core` crate has a `test-helpers` feature for test utility functions
- Repository pattern: create `SqliteXxxRepository` structs that take `Arc<Mutex<Connection>>` via `new()` ‚ÄĒ clone from `db.connection()`
- Enums are stored as snake_case TEXT in SQLite (matching serde `rename_all = "snake_case"`) ‚ÄĒ use manual serialize/deserialize helpers, not Display
- Clippy enforces `-D warnings` ‚ÄĒ use function references (`row_to_node`) instead of closures (`|row| row_to_node(row)`)
- `serde_json` is needed in seshat-storage for ext_data/metadata JSON columns ‚ÄĒ add as workspace dependency
- Add `seshat-core = { workspace = true, features = ["test-helpers"] }` to `[dev-dependencies]` for test factory functions
- For BLOB columns (e.g., `ir_data`), use `serde_json::to_vec` / `serde_json::from_slice` to serialize/deserialize entire structs
- Use `ON CONFLICT ... DO UPDATE SET` for upsert operations on tables with UNIQUE constraints
- Use `conn.unchecked_transaction()` for transactions within a `Mutex<Connection>` lock ‚ÄĒ avoids borrow issues
- `list_branches` gathers distinct branch_ids via UNION across nodes, edges, files_ir tables
- Current branch is stored in the `metadata` table under key `"current_branch"` ‚ÄĒ defaults to `"main"` if absent
- Config structs (ScanConfig, DetectionConfig, ServerConfig) in seshat-core need `#[serde(default)]` on the struct for partial TOML parsing to work
- In thiserror 2, naming a field `source` with a non-Error type (e.g., `String`) causes compile errors ‚ÄĒ use a different field name like `details`
- Clippy `-D warnings` flags manually implemented Default when it can be derived ‚ÄĒ use `#[derive(Default)]` when all fields have Default
- `toml` and `dirs` crates are workspace dependencies for config loading (added for US-004)
- Application-level config structs (WatcherConfig, BackupConfig, CacheConfig, EmbeddingConfig) live in `seshat-bin/src/config.rs`, not seshat-core
- CI workflows live in `.github/workflows/` ‚ÄĒ `ci.yml` for PR checks, `lint-workflows.yml` for actionlint
- `build.rs` in seshat-bin sets `GIT_HASH` env via `cargo:rustc-env` ‚ÄĒ access in code with `env!("GIT_HASH")`
- Use `cargo:rerun-if-changed=../../.git/HEAD` in build.rs to invalidate on new commits
- Release workflow uses `release-plz/action@v0.5` for automated version bumps, changelogs, and GitHub releases
- Pre-commit hooks are configured in `.pre-commit-config.yaml` ‚ÄĒ local hooks use `language: system` with `pass_filenames: false` for cargo commands
