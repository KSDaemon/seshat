## Codebase Patterns
- Detectors crate uses `seshat_core::{CodeEvidence, FunctionCall, Import}` imports
- Test helpers follow pattern: `make_<type>(params)` functions returning core types
- Tests use table-driven style with descriptive `#[test]` function names
- `cargo clippy` runs with `-D warnings` — all warnings are errors
- `cargo fmt` runs as pre-commit hook — code must be formatted
- clippy `manual_contains` lint: use `vec.contains(&val)` instead of `vec.iter().any(|&x| x == val)`
- When replacing evidence gathering with `find_usage_evidence_for_file`, always provide import/dep evidence fallback when call_sites is empty
- To add call-site evidence to framework findings in test_patterns, use same pattern: `let call_sites = find_usage_evidence_for_file(file, MAX_EVIDENCE); let evidence = if !call_sites.is_empty() { call_sites } else { /* fallback */ };`
