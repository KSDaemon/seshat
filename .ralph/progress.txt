## Codebase Patterns
- Use `refinery::embed_migrations!("migrations")` in `db.rs` to embed SQL migrations at compile time
- `Database` struct wraps `Arc<Mutex<Connection>>` — use `db.connection().lock().unwrap()` to get write access
- Migration files go in `crates/seshat-storage/migrations/` with naming `V{N}__{description}.sql`
- Core types (NodeId, EdgeId, BranchId, KnowledgeNode, Edge, etc.) are in `seshat-core` — import via workspace dependency
- All ID types use newtype pattern: `NodeId(pub i64)`, `EdgeId(pub i64)`, `BranchId(pub String)`
- `StorageError` in `seshat-storage/src/error.rs` is the crate-level error type — add variants there for new failure modes
- Use `":memory:"` for in-memory SQLite in tests (fast, no cleanup needed for simple cases)
- Workspace uses `thiserror = "2"` (not v1) for error derive macros
- The `seshat-core` crate has a `test-helpers` feature for test utility functions
