## Codebase Patterns
- Use `crate::db::detect_branch(path)` for unified branch detection — handles normal repos, worktrees, and detached HEAD
- Watcher `start_watcher` captures `branch_id` at call time; bulk rescan callback uses the captured value instead of re-detecting
- `get_current_branch(path)` returns `Option<String>` for raw HEAD reading; `detect_branch(path)` wraps it with `"main"` fallback + tracing::debug
- `resolve_project()` reads `project_root` from `repo_metadata` in the DB (via `read_project_root_from_db`); always pass this through to callers — never rely on `db_path.parent()` for branch detection because DBs live in XDG data dir, not at project root
- To inject system-level metadata into all MCP tool responses, mutate the serialized JSON in `execute_tool()` via `serde_json::Value` parsing rather than threading state into every tool handler's `ResponseMetadata`
