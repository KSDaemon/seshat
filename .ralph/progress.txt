## Codebase Patterns
- Use `gix` (v0.72) for git operations — already workspaced and used in seshat-graph
- Response data types derive `#[derive(Debug, Clone, Serialize)]`, use `#[serde(skip_serializing_if = "Option::is_none")]` on optional fields
- MCP tool modules live in `crates/seshat-mcp/src/tools/`, registered in `tools/mod.rs`, with `impl_tool_request!` in `server.rs`
- All tool handlers wrap results in `ResponseEnvelope::success` / map errors via `map_graph_error()`
- Enums use `#[serde(rename_all = "snake_case")]` for serialization
- Internal query helpers in `GraphError::query()` use `format!()` for error messages
- Tests use `tempfile::tempdir()` for isolated git repos, `Command::new("git")` for git operations
- `load_branch_ir(conn, branch_id)` in `code_pattern.rs` loads IR from files_ir table, shared by graph modules
- dependency analysis uses suffix matching at path boundaries to handle absolute vs relative paths
- Blast radius classification uses same thresholds everywhere: <=2 low, 3-10 medium, >10 high
