## Codebase Patterns
- Workspace monorepo: 10 crates under `crates/`; `seshat-bin` is the thin binary, `seshat-cli` is the CLI library
- CLI commands are organized as one module per file in `seshat-cli/src/`, declared in `lib.rs`, dispatched via `match cli.command` in `run()`
- Error handling: `thiserror`-based `CliError` enum, NOT `anyhow`
- Paths: `dirs` crate for XDG resolution; DBs at `$XDG_DATA_HOME/seshat/repos/`, cache at `$XDG_DATA_HOME/seshat/version-check.json`
- Testing: `#[cfg(test)] mod tests` inline unit tests; integration tests in `tests/` dirs using `assert_cmd` + `predicates`
- Serialization: `serde` + `serde_json` (already available workspace deps)
- Timestamps: `chrono` with `clock` feature (already available workspace dep)
- Commit hooks: `cargo fmt` and `cargo clippy` run via pre-commit
