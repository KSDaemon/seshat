## Codebase Patterns
- MCP tool handlers are thin: parse input, call seshat-graph, wrap in ResponseEnvelope/ErrorEnvelope
- All MCP response types use `serde::Serialize` and follow envelope pattern (status, tool, repo, branch, scope, data, metadata)
- ErrorCode enum in envelope.rs uses `#[serde(rename_all = "SCREAMING_SNAKE_CASE")]` with matching `as_str()` method
- Test helpers live in `lib.rs` under `#[cfg(test)] pub(crate) mod test_helpers`
- Module declarations in lib.rs are alphabetically ordered
- `serde_json::Value` is the standard way to represent dynamic JSON data
- `tempfile` crate available as workspace dependency for temp dirs in tests
- CallLogger API: `new(path)`, `session_id()`, `next_seq()`, `log_call(&entry)`
- McpServer::new() accepts `call_log_path: Option<PathBuf>` as 4th parameter
- All MCP request types derive `serde::Serialize` in addition to `Deserialize` (needed for call log input capture)
- `ExampleInput` in `tools/mod.rs` also derives `Serialize`
- Call log integration pattern: `self.call_logger.as_ref().map(|_| (input, Instant::now()))` — zero overhead when None
- `start_stdio` and `start_stdio_with_shutdown` accept `call_log_path: Option<PathBuf>` before shutdown/drain params
- Tool handler result summaries are extracted by parsing the response JSON string, not from raw Rust structs
- Error code is extracted from `parsed["error"]["code"]` in the response JSON
- `days_to_ymd()` helper in server.rs implements Hinnant's civil calendar algorithm for ISO 8601 timestamps without chrono
- CLI optional-value flags use clap `num_args = 0..=1` + `default_missing_value = ""` with `Option<PathBuf>` — `Some("")` = bare flag, `None` = not passed
- `ServerConfig.call_log` is `String` (empty = disabled) — simpler serde default than `Option<String>`
- `dirs` crate v6 is a workspace dependency for platform-specific directories
- `#[tool_router]` macro methods are private — integration-level call log tests go in `server.rs` `mod tests`, not a separate `tests/` dir

## 2026-04-04 19:05 (EEST) - US-002
- What was implemented
  - Added `CallLogger` struct to `crates/seshat-mcp/src/call_logger.rs` with fields: `writer: Mutex<BufWriter<File>>`, `session_id: String`, `seq: AtomicU64`
  - `CallLogger::new(path)` — creates parent dirs via `fs::create_dir_all`, opens file in append mode, generates 8-char hex session ID from system time hash (`DefaultHasher` on `SystemTime::now().as_nanos()`)
  - `log_call(&self, entry)` — acquires mutex, serializes to JSON string, writes line + newline, flushes buffer
  - `next_seq(&self)` — `fetch_add(1, Ordering::Relaxed)` starting at 0
  - `session_id(&self)` — returns `&str` reference to the session ID
  - `Debug` impl that shows session_id and current seq
  - `generate_session_id()` private helper using `DefaultHasher` on nanosecond timestamp
  - 6 unit tests: file creation, parent dir creation, JSONL validity, monotonic seq, session ID format, append behavior across two loggers
  - Added `tempfile.workspace = true` to `[dev-dependencies]` in `crates/seshat-mcp/Cargo.toml`
- Files changed
  - `crates/seshat-mcp/src/call_logger.rs` (added CallLogger struct, methods, and 6 tests)
  - `crates/seshat-mcp/Cargo.toml` (added tempfile dev-dependency)
- **Learnings for future iterations:**
  - Session ID is 8 lowercase hex chars (0-9, a-f) — subset of alphanumeric, generated from `DefaultHasher` on `SystemTime::now().as_nanos()`
  - `CallLogger` is `Send + Sync` — `Mutex<BufWriter<File>>` + `AtomicU64` + `String` are all thread-safe
  - `log_call` uses `serde_json::to_string` (not `to_writer`) to avoid partial writes on serialization errors
  - For US-003 integration: construct `CallLogger` with `CallLogger::new(path)`, get session via `session_id()`, get seq via `next_seq()`, log via `log_call(&entry)`
  - The `writer` mutex lock uses `.expect("call-log mutex poisoned")` — panics on poisoned mutex (deliberate: log corruption is unrecoverable)
