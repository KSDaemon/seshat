## Seshat — Project Convention Intelligence

ALWAYS use Seshat MCP tools BEFORE writing code.

| When | Call |
|------|------|
| Session start | `query_project_context` |
| Before new symbol | `query_code_pattern(query="<name>")` |
| Before any pattern | `query_convention(topic="<area>")` |
| Before writing | `validate_approach(description="<plan>")` |
| Before editing file | `query_dependencies(path="<file>")` |
| New pattern found | `record_decision(description="<what>", reason="<why>")` |

Load the `seshat` skill for full workflow details and examples.
